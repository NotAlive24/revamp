There are four main cloud deployment models: **public, private, hybrid, and community**. The deployment model affects how resources are shared and who is responsible for managing them.

**Public cloud** is provided by an external cloud service provider and is shared among many users. Organizations usually rent or lease these services.

**Private cloud** is used by a single organization. It may be built and managed by that organization or hosted with outside support, but it is dedicated to one organization only.

**Hybrid cloud** combines two or more cloud models, usually private and public. It allows organizations to keep critical workloads under tighter control while using public cloud for flexibility, scale, or cost savings.

**Community cloud** is designed for a specific group of organizations or users with similar needs or interests. Resources are shared to support that community’s common requirements.

**Main Idea**  
Cloud deployment models define how cloud resources are owned, shared, and managed.
